二、软件说明书

1. 引言

1.1 编写目的
本文档用于说明儿童专注力训练游戏软件 V1.0 的开发背景、功能范围、系统结构、运行环境、核心模块、算法流程、数据结构、操作流程和技术特点，可作为计算机软件著作权登记申请中的软件说明书、设计说明书和用户手册基础材料。

1.2 软件背景
儿童专注力训练需要兼顾趣味性、反馈即时性和训练连续性。传统训练方式往往依赖单一题目或人工观察，难以在训练过程中实时反映儿童的注意状态。本软件将游戏化任务与脑电专注度、情绪状态数据相结合，通过关卡机制和可视化反馈引导儿童完成持续注意、反应控制、路径规划、空间判断和情绪稳定训练。

1.3 软件适用对象
本软件适用于儿童注意力训练、认知训练、课堂辅助训练、家庭训练和机构训练场景。软件本身用于训练和交互反馈，不作为医学诊断、治疗或临床评估结论输出工具。

2. 软件总体概述

2.1 软件定位
本软件是一套以儿童专注力训练为核心目标的桌面游戏软件。用户通过登录进入系统，在关卡选择界面选择训练内容；系统根据游戏操作、关卡完成情况、专注度数据和情绪状态数据，对角色能力、关卡道具、提示反馈和难度表现进行动态调整。

2.2 软件组成
软件由 Godot 工程主体、脚本模块、主场景文件、角色与道具场景、关卡场景、素材资源、用户数据文件、答案配置文件、脑电算法模块、情绪算法模块、算法接入脚本和 42 个扩展关卡组成。

2.3 当前工程目录覆盖范围
1. project.godot：Godot 工程配置、主场景、输入映射、显示配置和自动加载脚本配置。

3. main_scene：标题、登录、设置、关卡选择、跑酷、搭桥草地关卡、搭桥城市关卡等主场景。
4. element：角色、平台、障碍、按钮、旗帜、弹簧垫、反向垫、升降平台等可复用场景组件。

3. 系统架构

3.1 架构分层
1. 表现层：标题界面、登录界面、设置界面、关卡选择界面、训练关卡界面、生命值显示、提示显示、角色动画和平台预览。
2. 游戏逻辑层：跑酷移动、跳跃、碰撞、生命值、平台生成、平台删除、答案提示、关卡切换、暂停和重置。
3. 生物反馈算法层：脑电数据读取、专注度指数计算、情绪状态识别、数据平滑、阈值判断和训练反馈映射。
4. 数据层：用户账号、本地档案、专注度基线、通关尝试次数、游玩时长、搭桥答案坐标、关卡配置和算法输出缓存。
5. 资源层：角色资源、地形资源、UI 资源、音效、字体、地图、场景预制件和扩展关卡资源。

3.2 主流程
1. 启动软件，Godot 加载主场景和自动加载的专注度模块。
2. 用户进入标题界面，点击开始进入登录或关卡选择流程。
3. 用户登录或创建账号，系统读取或初始化用户档案。
4. 用户选择训练关卡，系统加载对应场景、道具和关卡配置。
5. 脑电与情绪算法接入脚本持续写入或推送状态数据。
6. 游戏脚本读取专注度和情绪状态，根据阈值改变角色能力、平台透明度、碰撞状态、提示触发和反馈效果。
7. 用户完成关卡后，系统记录尝试次数、训练时长和必要的通关结果。
8. 用户可返回关卡选择界面继续训练或退出软件。

4. 功能模块说明

4.1 标题与导航模块
对应场景包括 title.tscn、subtitle.tscn、setting.tscn、parkour.tscn 及相关 start_button 脚本。该模块负责软件启动后的标题展示、开始游戏、关卡入口、返回上级界面、设置界面跳转和场景切换。

4.2 用户登录与档案管理模块
对应 login_controller.gd、user_data_store.gd 和 users.json。该模块提供用户名密码登录、用户创建、当前用户设置、专注度基线保存、关卡成功尝试次数保存、累计游玩时长保存和用户数据规范化处理。数据优先保存在 Godot user:// 目录，同时在开发阶段同步到项目 data/users.json，便于调试和归档。

4.3 专注度读取与脑电接入模块
对应 concentration.gd 及额外脑电接入脚本。当前工程中的 concentration.gd 以自动加载节点形式运行，按固定间隔读取本地 concentration.txt 中的专注度数值，并将其限制在 0 至 100 之间。脑电接入脚本负责接收脑电设备或算法服务输出的数据，完成数据清洗、特征处理、专注度指数计算和标准化输出，再将结果写入本地文件、JSON、Socket、HTTP 接口或 Godot 可读取的数据通道。

4.4 情绪算法模块
情绪算法模块用于接入儿童训练过程中的情绪状态数据。算法可根据实际设备来源读取面部表情、语音、行为、生理或脑电衍生特征，经过归一化、去噪、滑动窗口平滑、特征融合和分类映射，输出平静、积极、紧张、疲劳、焦虑、低唤醒等情绪标签或情绪分值。游戏逻辑根据情绪状态调整提示频率、反馈强度、关卡节奏和训练难度，避免训练过程过度挫败或刺激不足。

4.5 跑酷训练模块
对应 player_parkour.gd、parkour_random_loop.gd、platform_elevator_grass.gd、platform_elevator_city.gd、reverse_direction_pad.gd、spring_pad.gd、heart_life_display.gd 等。模块支持角色自动奔跑、手动移动、跳跃、重力、碰撞、障碍扣血、掉落回退、生命值显示、弹簧垫、反向垫、升降平台、随机平台循环和终点跳转。专注度达到设定阈值时，角色可进入无敌或飞行状态，并提高移动速度。

4.6 搭桥训练模块
对应 platform_spawn.gd、bridge_build_answer_hint.gd、bridge_build_answers.json、bridgebuildgrass1.tscn 至 bridgebuildgrass10.tscn、bridge_build_city_1.tscn 至 bridge_build_city_10.tscn。用户通过鼠标或键盘在网格位置生成平台，系统检测平台数量、重叠、障碍碰撞和合法放置区域。平台的透明度和碰撞状态与专注度阈值绑定，专注度越高，平台越接近实体状态。答案提示模块支持记录答案坐标、读取关卡答案、按 P 键或专注度条件触发提示闪烁。

5. 核心算法说明

5.1 脑电专注度算法流程
1. 数据采集：通过脑电设备 SDK、串口、蓝牙、USB、WebSocket、HTTP 或本地文件读取脑电原始数据、设备注意力指标或频段特征。
2. 数据预处理：对异常值、丢包、噪声、眨眼干扰、肌电干扰和短时突变进行过滤或标记。
3. 特征提取：提取注意力相关指标，包括但不限于频段能量、稳定度、信号质量、连续有效时长、波动幅度和设备输出的注意力指数。
4. 指数融合：将多项特征按权重融合为 0 至 100 的专注度指数，并根据儿童个人基线进行归一化。
5. 平滑处理：采用滑动窗口、指数平滑或中值滤波，减少瞬时波动对游戏反馈的影响。
6. 阈值映射：将专注度指数映射为低专注、中等专注、高专注和极高专注等状态。
7. 游戏联动：低专注时平台呈轮廓或半透明状态；中等专注时平台逐步成形；高专注时平台获得实体碰撞；更高专注时角色可触发无敌、飞行、加速或额外提示奖励。

5.2 情绪识别算法流程
1. 数据输入：根据实际部署接入面部表情、语音语调、行为操作、生理信号或脑电衍生特征。
2. 特征标准化：将不同来源的情绪特征转换为统一量纲，处理缺失数据和异常数据。
3. 状态识别：通过规则模型、机器学习模型或深度学习模型输出情绪标签或情绪分值。
4. 时间平滑：对连续帧或连续采样窗口的情绪结果进行平滑，避免状态频繁跳变。
5. 可信度判断：当输入质量不足或模型置信度较低时，保持上一稳定状态或切换为未知状态。
6. 游戏反馈：根据情绪状态调整提示密度、奖励音效、视觉反馈、关卡速度、失败惩罚强度和休息建议。

5.3 算法接入脚本说明
算法接入脚本用于连接外部设备、算法服务和 Godot 游戏主体。接入脚本应包括以下职责：
1. 设备连接：初始化脑电设备、摄像头、麦克风或算法服务连接。
2. 数据接收：按固定采样周期接收原始数据或算法结果。
3. 格式转换：将外部数据转换为游戏可读取的标准字段，如 focus_level、emotion_label、emotion_score、signal_quality、timestamp。
4. 数据输出：通过本地文本、JSON、Socket、HTTP 或 Godot 插件接口输出给游戏。
5. 异常处理：处理设备断开、数据为空、信号质量差、算法超时和权限不足等情况。
6. 日志记录：记录接入状态、采样时间、算法输出和异常信息，便于调试和追溯。

5.4 训练反馈映射规则
1. 专注度低于 40：平台仅显示轮廓或低透明度，提示儿童集中注意。
2. 专注度 40 至 60：平台半透明，允许儿童观察即将形成的位置。
3. 专注度 60 至 75：平台逐步成形，给予正向反馈。
4. 专注度 75 及以上：平台实体化并启用碰撞；角色可触发高专注能力或关卡奖励。
5. 情绪紧张或疲劳：降低关卡节奏，增加提示，弱化失败惩罚。
6. 情绪稳定或积极：维持正常训练节奏，适度提高挑战密度。

6. 数据结构说明

6.1 用户数据 users.json
字段包括 schema_version、users、username、password、focus_baseline、level_success_attempts 和 total_play_time_seconds。该结构用于保存用户档案和训练记录。

6.2 搭桥答案 bridge_build_answers.json
字段以关卡场景路径为键，以平台答案坐标数组为值。每个坐标记录一个推荐平台放置位置，用于答案提示和关卡验证。

6.3 专注度数据 concentration.txt
文件中保存 0 至 100 的专注度整数或浮点数。游戏自动加载模块按间隔读取该文件，更新全局专注度状态。

6.4 算法状态数据
额外脑电与情绪算法接入脚本建议输出如下字段：focus_level、focus_raw、focus_baseline、emotion_label、emotion_score、signal_quality、device_status、timestamp、source。该数据可保存为 JSON 或通过实时通信接口传入游戏。

8. 用户操作说明

8.1 启动软件
用户运行软件后进入标题界面，点击开始按钮进入登录或训练流程。系统初始化输入映射、显示窗口、专注度自动加载节点和主场景。

8.2 登录与创建用户
用户输入用户名和密码后点击登录。若账号不存在，可点击创建用户。系统保存用户名、密码、专注度基线、关卡尝试记录和累计训练时长。

8.3 选择训练关卡
用户在关卡选择界面选择跑酷、搭桥或扩展训练关卡。软件加载对应场景和脚本，并读取关卡配置、答案数据和算法状态。

8.4 跑酷训练操作
用户可通过键盘控制跳跃或移动。角色在关卡中自动或手动前进，遇到障碍会扣除生命值。专注度较高时角色可获得无敌、飞行或加速反馈。生命值耗尽后关卡重置。

8.5 搭桥训练操作
用户通过鼠标左键或键盘在网格上放置平台，通过鼠标右键删除平台。系统检查平台数量、位置合法性和碰撞状态。专注度达到阈值后平台由透明逐渐变为实体。用户可按提示键查看答案提示。

8.6 脑电与情绪数据接入
脑电设备或情绪识别服务启动后，接入脚本将算法结果转换为统一字段并写入游戏可读取通道。游戏读取专注度和情绪状态后实时更新关卡反馈。

8.7 暂停、设置与退出
用户可通过暂停或设置界面中断训练、调整流程、返回关卡选择或退出软件。系统根据当前用户记录必要训练信息。

9. 技术特点

1. 游戏化训练：以跑酷、搭桥和扩展关卡替代单一训练题目，提高儿童参与度。
2. 生物反馈闭环：脑电专注度和情绪状态可实时影响游戏机制，形成“状态采集、算法识别、游戏反馈、继续训练”的闭环。
3. 阈值可配置：专注度阈值、平台透明度、碰撞启用、飞行阈值、无敌阈值和速度倍率均可通过脚本参数配置。
4. 模块化场景：Godot 场景与脚本分离，角色、道具、平台、关卡和 UI 组件可复用。
5. 本地数据管理：用户档案、训练记录、答案坐标和算法状态可在本地保存，便于离线训练。
6. 关卡可扩展：系统支持在既有关卡基础上继续增加扩展关卡，并复用平台生成、答案提示、专注度映射和情绪反馈逻辑。
7. 训练反馈清晰：生命值、平台透明度、角色状态、提示闪烁、场景切换和奖励效果共同形成即时反馈。
8. 儿童友好：采用像素风角色、地图和 UI，交互方式直观，适合低龄用户理解和操作。
